# Gestión del Proyecto: Luciano's Boutique

Para el desarrollo de **Luciano's Boutique**, estamos organizando el trabajo mediante un tablero Kanban en **Trello**. Esto nos permite gestionar visualmente el flujo de tareas y asegurar un desarrollo continuo y ordenado.

## Estructura del Tablero Kanban

Hemos dividido el tablero en 4 columnas principales:

1. **Lista de tareas (To Do):** Contiene el backlog de tareas pendientes por realizar, incluyendo requerimientos de Frontend, Backend, Estilos, entre otros.
2. **En proceso (In Progress):** Tareas en las que se está trabajando activamente en este momento.
3. **Review:** Tareas cuyo desarrollo ha finalizado, pero están pendientes de revisión (Code Review, Testing) antes de integrarse definitivamente.
4. **Done:** Tareas completamente terminadas, aprobadas y funcionales.

## Categorización de Tareas

Las tareas (tarjetas) se clasifican principalmente en:
- **Frontend:** Creación de componentes, páginas, hooks, e integración con APIs.
- **Backend:** Diseño de rutas, controladores, servicios y configuración del servidor.
- **Estilos:** Configuración y aplicación de Tailwind CSS, diseño responsivo y UI/UX.

Este enfoque nos ayuda a mantener claridad en el progreso del proyecto y facilitar la colaboración.
